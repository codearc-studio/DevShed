Features

Designed to Be Simple and Safe

DevShed safely handles complex storage sweeps in one click, without risking your critical source directories.

Find Hidden Cache

Xcode stores huge builds even after you delete or move the project file. DevShed automatically flags these abandoned folders so you can delete them cleanly.

Choose What to Clean

Prevent compile slowdowns. Use the interactive planner to inspect files, view sizes, and select only what you want to wipe.

Safe Cleaning

DevShed asks you to save open projects and closes Xcode automatically before starting a cleaning cycle, keeping your work secure.

Project Folder Scanner

Monitor external work folders to track down massive and obsolete dependencies that clutter your storage.

Simulator Cleanup

Safely clear heavy simulator system files and caches generated during testing to free up local disk space.

Smart Alerts

Set a storage warning threshold. Let DevShed alert you when hidden Xcode caches get too large, so your drive stays clean.

Purge Targets

Clean Everything Safely

Xcode hoards storage across multiple folders. DevShed organizes these areas cleanly, allowing you to choose exactly what to clear.

Package Cache Protection: Excludes core registries by default to save network bandwidth and avoid slow re-downloads on your next project load.

Derived Data & Archives

Removes build directories, intermediate compilation folders, logs, and expired local release archives.

Simulator Logs & Caches

Safely discards heavy testing diagnostic reports, console log files, and obsolete simulator device support data.

Multi-Platform Package Cache

Direct support to clean nested package folders like Flutter Pub, CocoaPods local files, Swift Package Manager, and Carthage.

FAQ

Frequently Asked Questions

Absolutely safe. DevShed never touches or deletes your actual source code or assets. It only clears out compiler cache, logs, and simulator directories that macOS automatically recreates when you build or test your app again.
When you delete an old project folder, Xcode often keeps gigabytes of index files behind inside your Library path. DevShed detects these abandoned directories that no longer correspond to a project on your drive, classifying them as Ghost Caches so you can clean them up.
Zero. We do not run background telemetry, metrics, or usage analysis. All file scans and deletions happen entirely inside your local macOS system environment.
