Features

Engineered for High-Performance Workspaces

DevShed is more than a directory eraser. Written natively, it targets exact diagnostic registries, packages, and redundant virtual states to keep compile times low and free storage high.

Ghost Cache Tracking

Xcode stores huge builds even after you delete or relocate the workspace file. DevShed queries original file logs to spot these orphaned, heavy folders and flags them as Ghost Caches for painless deletion.

Interactive Purge Planner

Prevent accidental rebuild slowdowns. Use the built-in Interactive Purge Planner to inspect files, view sizes down to individual paths, and selectively exclude mission-critical targets prior to wiping.

Execution Safe Mode

Avoid index corruption. When active, Safe Mode prompts you to save current project files and automatically closes running Xcode processes gently before launching any deletion routine.

Node Modules Watcher

Add custom developer roots. Locate, catalog, and clean out heavy, nested local `node_modules` folders hidden deep inside neglected directories or expired project archives.

Simulator Device Tracker

Review simulated active targets directly. Safely clear device runtime containers, clean stale log states, and erase legacy iOS simulator versions abandoned by Apple system revisions.

Active Auto Pilot

Unattended storage maintenance. Set your optimal threshold limits, and let DevShed safely wipe expired caching repositories dynamically in the background without disturbing active flows.

Multi-Platform & Native Support

Xcode generates different types of caching. DevShed maps out all developer cache directories natively—giving you full transparency of what can be cleaned.

Swift SPM Cache Protection: Excludes core SPM artifact registries by default to preserve compiler indexes and save internet bandwidth on your next build.

Derived Data & Archives

Clears intermediate compilations, module maps, debug configurations, and redundant App Store release archives securely.

iOS Device Support symbols

Safely clears symbolic metadata mapped from old hardware connections, complete with instant backup and restoration tools built in.

Multi-Platform Cache Catalogs

Direct support for clearing package folders including Flutter Pub Cache, Homebrew formula binaries, Carthage, CocoaPods databases, and Swift PM dependencies.

Frequently Asked Questions

Absolutely safe. DevShed never touches or views your personal source files or project workspaces. It targets only compiler cache output repositories, logs, intermediate object maps, and temporary simulators which macOS is engineered to dynamically regenerate on demand.
When you delete a folder containing an `.xcodeproj` or `.xcworkspace` file, Xcode continues to hold gigabytes of compilation index files inside your global `DerivedData` directory. DevShed tracks the original file references, recognizes that the host project folder no longer exists on disk, and classifies these abandoned assets as "Ghost Caches" so you can clean them with peace of mind.
Wiping intermediate compile packages completely sometimes deletes the package manager cached dependency checkouts, meaning your next build requires a full re-download. When DevShed's SPM Protection is enabled, we surgically exclude directories containing dependency downloads so post-clean compilations remain blisteringly fast even offline.
Zero. Absolutely none. We do not track what you wipe, what your projects are called, or even how many times you run the app. All path inspections and purges occur purely sandboxed on your local macOS workstation.
